# B-hydra Core — заметки для разработки (handoff)

Этот файл — контекст проекта для любой Claude-сессии (включая Fable 5).
Прочти его перед изменениями: здесь что уже сделано, как оно работает и какие
инварианты нельзя ломать, иначе B-hydra перестанет работать/синхронизироваться.

B-hydra — учебная P2P-криптовалюта на Python (UTXO, PoW, своя SHA-2 и ECDSA).
Ядро — пакет `b_hydra/`. Десктоп-клиент — «B-hydra Core» (`bhydra_gui.py`, tkinter).

## 🔒 ГЛАВНОЕ ПРАВИЛО (безопасность) — не нарушать
**Приватный ключ НИКОГДА не покидает устройство.** Транзакция подписывается
локально (у клиента), на узел уходит уже ПОДПИСАННАЯ транзакция (vin с
public_key + signature, но без приватного ключа) через `POST /api/transaction`.
Эталон — `b_hydra/mobile_client.py`.

НЕЛЬЗЯ: слать `private_key` на сервер и подписывать на узле. Эндпоинты вроде
`/api/send` или `/api/wallet`, принимающие `private_key`, — это кастодиальная
модель и утечка ключа (API слушает 0.0.0.0, CORS `*`, обычный HTTP → ключ
уходит открытым текстом). Если делаешь веб-кошелёк — подписывай в браузере
(JS ECDSA secp256k1 по схеме из `API.md`) и шли только подпись.

## Инварианты консенсуса (сломаешь — узлы не синхронизируются)
- Хеш по умолчанию — СВОЯ реализация SHA-2 «с нуля» (`b_hydra/sha2.py`, без
  hashlib). Переключатель: env `BHYDRA_PURE_SHA=0` или `hashing.use_pure_sha(False)`.
  Значения битово совпадают с hashlib; майнить «по-настоящему» надо на pure.
- PoW: `int(hash,16) <= target`. Работа блока = `2**512 // target`.
- Генезис детерминирован (фикс. timestamp 0.0 и `genesis_target` из difficulty).
  Разная базовая difficulty → разный генезис → «другая сеть», sync отвергается.
- Сложность: биткоин-ретаргетинг по времени (`Blockchain.expected_target`),
  окно `RETARGET_INTERVAL` (100), изменение ≤ `MAX_ADJUST_FACTOR` (4) за
  пересчёт, не легче генезиса. Всё целочисленно и детерминировано из timestamps.
- Выбор главной цепи — по СУММАРНОЙ работе (`total_work`), не по длине.
- Replay-защита: `CHAIN_ID` вшит в подпись транзакции.
- Лимит блока: `MAX_BLOCK_TRANSACTIONS = 5000` (≈3.7 МБ).
- Скриптовые выходы UTXO: `TxOutput.script` (None → обычный P2PKH; dict → условие).
  Первый тип — HTLC (hash+time lock), проверяется в `node._authorize_input` /
  `_eval_htlc` с учётом высоты блока. Обычные выходы сериализуются БЕЗ поля
  `script` — иначе изменятся txid и сломается совместимость. `get_balance` /
  `find_spendable` пропускают скриптовые выходы. `TxInput.preimage` — witness
  (не входит в подпись/txid). Трест-лесс смарт-чек: `contract.fund_cheque` /
  `redeem_cheque_onchain` / `refund_cheque_onchain`.

## Экономика (параметры жёстко связаны!)
`INITIAL_REWARD=50`, `HALVING_INTERVAL=310_000`, `MAX_SUPPLY=31_000_000`,
`DECIMALS=8`, `DEFAULT_FEE=0.0001`. Из них следует время блока ~48.6 мин и конец
эмиссии ~3000 год (пасхалка «Футурама»). Менять одно — пересчитывать остальные:
`31_000_000 = 310_000 * 50 * 2`. 10-мин блок несовместим с «до 3000 года» при
награде 50 и кэпе 31M — это осознанный выбор пользователя (оставить 3000).

## Что уже сделано в основной ветке
- Кошелёк: ключи ECDSA secp256k1 (`wallet.py`), Base58-адреса с checksum,
  терпимый импорт ключа (0x/пробелы/регистр) с понятными ошибками.
- Десктоп GUI (`gui.py`): вкладки Кошелёк/Майнинг/Сеть/Блоки/Мемпул;
  копирование адреса, QR-код (свой генератор `qrcode_gen.py`), комиссия,
  история операций (Пополнение/Отправка/Себе/Майнинг, от кого/куда/когда,
  двойной клик → обозреватель), авто-майнинг (блок раз в N минут).
- P2P (`p2p.py`): рассылка блоков/tx, sync по работе, автосинк, seed-узлы,
  обмен пирами, авто-поиск в LAN по UDP-маякам (WiFi без интернета), бинд на
  0.0.0.0, `_BoundedSet` для seen_tx/seen_blocks (без утечки памяти).
- Экономика (`economics.py`), обозреватель (`explorer.html`), REST API (`api.py`).

## Запуск
    python bhydra_gui.py                 # десктоп B-hydra Core (нужен tkinter)
    python cli.py wallet | mine | send   # CLI
    python api.py --port 8000            # REST API + веб-обозреватель
    python P2P.py --demo                 # демо P2P из 3 узлов
Готовый `.exe` — в GitHub Releases (сборка `.github/workflows/build.yml` по тегу).
